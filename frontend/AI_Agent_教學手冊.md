## Unit 0：基礎概念教學

### 0-1 什麼是 AI Agent？（概念說明）

**白話解釋：**  
AI Agent（AI 代理人，*agent* = 代理者）就像是一個會自己思考、自己動手的智慧助理。你給它一個目標，它會把任務拆解成多個步驟並逐一完成，中途還能使用各種工具（搜尋網路、操作 App、發送訊息）。

**與一般 AI 聊天機器人（Chatbot）的差異：**

| 比較面向 | 一般 AI 聊天（Chatbot） | AI Agent |
|----------|------------------------|----------|
| 互動方式 | 問一句、答一句 | 接收目標、自主執行多步驟 |
| 工具使用 | 無（只有語言） | 可呼叫搜尋、日曆、訂票等工具 |
| 記憶能力 | 只記當次對話 | 可記住跨對話的偏好與進度 |
| 自主程度 | 被動等待 | 主動規劃並完成任務 |
| 代表產品 | 早期 ChatGPT | ChatGPT with Actions、AutoGPT |

**語法結構：寫給 AI Agent 的指令（Prompt）公式**

```
[角色設定 Role]  +  [背景資訊 Context]  +  [具體任務 Task]  +  [輸出格式 Format]
```

| 要素 | 說明 | 範例 |
|------|------|------|
| **Role**（角色設定） | 告訴 AI 扮演什麼角色 | 「你是我的旅遊規劃師」 |
| **Context**（背景資訊） | 提供你的條件與限制 | 「預算 3 萬，偏好文化景點」 |
| **Task**（具體任務） | 明確說明要做什麼 | 「規劃 5 天 4 夜京都行程」 |
| **Format**（輸出格式） | 指定回答的呈現方式 | 「用表格列出每日行程」 |

---

### 0-2 最小可執行範例

**❌ 模糊指令（效果差）：**
```
幫我規劃旅遊
```

**✅ 完整指令（效果好）：**
```
你是我的旅遊規劃師（Role）。
我是一對夫妻，無小孩，預算 3 萬台幣，偏好文化景點（Context）。
請規劃 5 天 4 夜的京都旅遊（Task）。
用表格呈現每日行程，附上住宿區域建議與預算分配（Format）。
```

---

### 0-3 常見錯誤與注意事項

| 常見錯誤 | 說明 | 修正方式 |
|----------|------|----------|
| 指令過於模糊 | AI 只能猜測你的需求 | 加入 Context 與 Format |
| 期待 AI 全知 | AI 會產生「幻覺」(Hallucination) | 重要資訊需自行查核 |
| 洩漏敏感資料 | 身份證、密碼不應輸入 | 用「XXX」或匿名化替代 |
| 只用一次就放棄 | AI 需要多輪對話調整 | 給予反饋讓 AI 改進 |

> 💡 **現在試試看：** 選一個你今天遇到的問題，試著用 Role + Context + Task + Format 四要素寫一個完整指令，貼給 ChatGPT 或 Claude 測試看看。

## Unit 1 — 個人行程規劃與時間管理

### 概念教學

**核心能力：** 把口語描述的行程轉化成結構化的時間表，並讓 AI 偵測衝突與提醒。

**關鍵技術術語：**
- **Scheduling**（排程）：安排事件的時間順序
- **Conflict Detection**（衝突偵測）：找出時間重疊的事項
- **Natural Language Processing, NLP**（自然語言處理）：AI 理解你的口語描述

**指令結構圖示：**
```
輸入：「我週三 2 點要開會，週三 3 點要接小孩放學」
  ↓
AI 解析時間資訊（NLP）
  ↓
偵測衝突（開會結束時間不確定）
  ↓
輸出：行程表 + 衝突警告 + 建議方案
```

---

### 練習題

**Easy — 練習 1：基本行程整理**

```
題目：
你這週的事項如下（請把它貼給 AI）：
- 週一：早上 10 點線上英文課（1 小時）
- 週三：下午 2 點要去郵局繳費
- 週四：晚上 7 點朋友聚餐（地點：台北東區）
- 週五：要交一份報告給老師（截止時間不明）

指令任務：
請 AI 幫你 (1) 整理成行事曆格式 (2) 找出資訊不完整的地方 
(3) 建議你需要提前準備什麼
```

**參考解答（指令寫法）：**
```
你是我的行程助理（Role）。
以下是我本週的事項：
- 週一早上 10 點：線上英文課（約 1 小時）
- 週三下午 2 點：去郵局繳費（距離住家約 10 分鐘）
- 週四晚上 7 點：朋友聚餐，台北東區
- 週五：需交報告給老師（截止時間未知）
（Context）

請幫我：
1. 整理成「日期 ｜ 時間 ｜ 事項 ｜ 備注」的表格格式
2. 標出資訊不完整需要確認的地方
3. 條列出每個事項需要提前準備的事（Format）
```

---

**Medium — 練習 2：衝突偵測與重新安排**

```
題目：
以下行程有衝突，請 AI 找出並提出解決方案：
- 週二下午 3 點：去銀行開戶（預計 1~1.5 小時）
- 週二下午 4 點：要接孩子放學（學校位置在銀行附近，步行 20 分鐘）
- 週二晚上 6 點：家庭晚餐（需要買菜，超市在回家路上）

提示：
在指令中說明「請列出可能的衝突」並要求 AI 提出「調整後的最佳安排」
```

**參考解答（指令寫法）：**
```
你是我的行程助理。
以下是我週二的行程，請幫我找出潛在的時間衝突：
- 下午 3 點：去銀行開戶（銀行地址：XX路XX號，預計 1~1.5 小時）
- 下午 4 點：接孩子放學（學校距銀行步行約 20 分鐘）
- 晚上 6 點：家庭晚餐（需要先去超市買菜，超市在回家路上）

請：
1. 列出所有可能的時間衝突
2. 提出調整後的「最佳行程順序」
3. 說明每個決策的理由
```

---

### ✏️ 學習重點

> - 越詳細的地點與距離資訊，AI 的排程越準確
> - 告訴 AI「截止時間」比「大概這週要做」更有效
> - AI 不會主動問你缺少的資訊，你需要在指令中包含所有條件

## Unit 2 — 購物比價與採購建議

### 概念教學

**核心能力：** 用結構化條件篩選商品，讓 AI 替代你做「比較」的腦力勞動。

**關鍵技術術語：**
- **Criteria-based Filtering**（條件篩選）：依需求過濾選項
- **Trade-off Analysis**（取捨分析）：在多個特性之間做出選擇
- **Recommendation Engine**（推薦引擎）：根據偏好排序建議

**採購決策框架：**
```
需求分析：我「真正需要」什麼？
    ↓
硬性條件：預算上限、必備功能（Must-have）
    ↓
軟性條件：偏好特性、加分選項（Nice-to-have）
    ↓
排除條件：不接受的特性（Dealbreaker）
    ↓
AI 比較輸出 → 你做最終決定
```

---

### 練習題

**Easy — 練習 1：商品採購條件整理**

```
題目：
你想買一台電風扇，請根據以下條件寫出完整的採購指令：
- 預算：2500 元以下
- 房間大小：約 5~6 坪小套房
- 最重要的：靜音（夜間使用）
- 希望有：搖頭功能
- 不需要：WIFI 智能功能（增加費用不需要）

任務：
用採購決策框架寫出指令，要求 AI 推薦 3 款並以表格比較
```

**參考解答：**
```
你是我的採購顧問（Role）。
我需要購買一台電風扇，條件如下（Context）：
- 預算：2500 台幣以下（硬性）
- 使用場景：約 6 坪小套房，主要夜間使用
- 必備：低噪音設計（分貝數越低越好）
- 希望有：左右搖頭功能
- 不需要：智能 WIFI 功能

請推薦 3 款適合的電風扇（Task），
用表格列出：品牌型號｜價格｜噪音（dB）｜特色｜適合理由（Format），
最後給出你的首推選項與原因（50 字以內）。
```

---

**Medium — 練習 2：防衝動購物分析**

```
題目：
請把下面的購物清單貼給 AI，要求它幫你分析是「真正需要」還是「衝動購物」：

- 第 4 個馬克杯（家裡已有 3 個）
- 新的工作滑鼠（舊的開始有點卡頓）
- 健身滾輪（計畫要開始健身）
- 植物香薰機（看到廣告覺得很療癒）
- USB 集線器（電腦連接埠不夠用）

提示：指令中要說明你目前的生活狀況，讓 AI 能做個人化判斷
```

**參考解答：**
```
你是我的消費行為顧問（Role）。
我的生活狀況：單身、住套房、上班族、每月可支配消費約 5000 元（Context）。
以下是我想買的清單，請幫我逐項分析（Task）：

1. 第 4 個馬克杯（家裡已有 3 個）
2. 新的工作滑鼠（舊的有點卡頓，用了 3 年）
3. 健身滾輪（計畫要開始健身）
4. 植物香薰機（看廣告覺得療癒）
5. USB 集線器（電腦連接埠不夠用）

請對每項分析：
- 判斷：✅ 合理購買 / ⚠️ 可以等等 / ❌ 建議不買
- 理由（一句話）
- 如果建議等等，告訴我什麼情況下才適合買（Format）
```

---

### ✏️ 學習重點

> - 「必備」(Must-have) 和「希望有」(Nice-to-have) 要分開說清楚
> - 說明「使用場景」比說「我要最好的」更能得到實用建議
> - AI 不會受廣告影響，善用它做客觀的取捨分析

## Unit 3 — 旅遊規劃與訂票

### 概念教學

**核心能力：** 從零規劃一趟完整旅遊，讓 AI 成為你的私人旅遊顧問。

**關鍵技術術語：**
- **Itinerary**（行程表）：按時間排列的旅遊計畫
- **Budget Allocation**（預算分配）：將總預算分配到各項目
- **Travel Style**（旅遊風格）：個人的旅行偏好類型

**完整旅遊指令必備要素：**
```
① 目的地 + 天數       → 決定行程密度
② 出發地 + 交通偏好   → 決定交通方式
③ 同行人數與特性      → 決定景點與餐廳類型
④ 總預算範圍          → 決定住宿等級與活動
⑤ 旅遊風格偏好        → 決定景點選擇邏輯
⑥ 特殊需求或排除項目  → 避免不適合的安排
```

---

### 練習題

**Easy — 練習 1：國內一日遊**

```
題目：
你住在台北，想週末去台南玩一天。
喜好：台灣小吃、老街、傳統建築
限制：搭高鐵，下午 6 點前要回到台北

請寫出完整指令，要求 AI 生成「含時間點的詳細行程」
```

**參考解答：**
```
你是我的國內旅遊規劃師（Role）。
我住在台北，計畫週六去台南一日遊（Context）：
- 出發方式：高鐵（台北 → 台南）
- 旅伴：一個人
- 喜好：台灣傳統小吃、老街、廟宇或歷史建築
- 回程限制：台南出發時間不超過下午 5:00（要在 6 點前回台北）

請幫我規劃一日行程（Task），格式如下（Format）：
| 時間 | 地點/活動 | 預估花費 | 交通方式 |
並在最後附上：
- 建議攜帶的東西
- 1 個省錢小技巧
```

---

**Medium — 練習 2：海外多日行程**

```
題目：
規劃一趟「日本大阪 4 天 3 夜」行程，條件自訂（填入你自己的偏好），
要求 AI 輸出：① 每日行程表 ② 預算分配表 ③ 注意事項清單

進階挑戰：
在 AI 給出行程後，用第二輪對話說：
「請把第二天的行程改成偏向美食探索，減少購物行程」，
觀察 AI 如何調整。
```

**參考解答：**
```
[第一輪指令]
你是我的日本旅遊專家（Role）。
我計畫大阪 4 天 3 夜的旅行（Context）：
- 出發地：台灣（飛關西機場 KIX）
- 旅伴：兩個人（情侶）
- 總預算：每人 25,000 台幣（含機票）
- 偏好：街頭美食、動漫文化、夜生活、拍照景點
- 不喜歡：美術館、過度觀光化的景點

請規劃完整行程（Task），包含（Format）：
① 四天每日行程表（含時間點、景點、餐廳推薦）
② 預算分配表（機票/住宿/餐飲/交通/購物各佔多少）
③ 5 點旅遊注意事項

[第二輪微調]
請把第二天的行程改成以「道頓堀美食探索」為主軸，
減少一個購物景點，並新增一個當地人才知道的隱藏餐廳推薦。
```

---

### ✏️ 學習重點

> - 多輪對話微調（Iterative Refinement）是 AI 旅遊規劃的核心技巧
> - 「同行人特性」是最容易被忘記但最重要的條件
> - AI 生成的行程仍需自行確認景點開放時間與票價

## Unit 4 — 文件撰寫與文字處理

### 概念教學

**核心能力：** 給 AI 關鍵素材，讓它產出高品質的文書文件。

**關鍵技術術語：**
- **Tone**（語氣）：文字傳遞的情感與正式程度
- **Paraphrasing**（改寫）：保留意思但改變表達方式
- **Summarization**（摘要）：壓縮長文為重點

**語氣（Tone）光譜：**

```
非正式 ←————————————————————→ 正式
  口語    友善    中性    專業    法律
  
例子：
口語：「那個問題修好了，你試試看」
友善：「很高興告訴您，問題已修復，歡迎重新測試」
專業：「茲通知您，相關技術問題已於本日完成修正」
```

---

### 練習題

**Easy — 練習 1：請假信撰寫**

```
題目：
情況：你的爸爸突然住院，需要請假陪伴照顧。
請假日期：明天到後天（2 天）
收件人：直屬主管（正式職場關係）

任務：寫出完整指令，讓 AI 產出一封語氣誠懇的繁體中文請假信（200 字以內）
```

**參考解答：**
```
請幫我寫一封正式請假信（Task），收件人是我的直屬主管（Context）：
- 請假原因：父親突然住院，需要陪伴照顧
- 請假日期：明天（週四）至後天（週五），共 2 天
- 工作交接：重要任務已交代同事代理
- 語氣要求：誠懇、正式但不過於冷漠，表達歉意
- 字數：200 字以內
- 語言：繁體中文（Format）
```

---

**Medium — 練習 2：會議記錄整理**

```
題目：
以下是一段雜亂的會議討論逐字稿片段（自己模擬一段），
請用 AI 整理成正式的會議記錄格式。

逐字稿範例（可自行替換）：
「好啦，那我們決定新系統上線時間往後推兩週好了，
等小明那邊的測試報告出來再說，
然後行銷那邊阿美說她要準備一個宣傳方案，
大概下週五之前給我們，對不對阿美？」

輸出格式應包含：會議主旨、決議事項、待辦（含負責人與期限）
```

**參考解答：**
```
以下是一段會議討論的逐字稿，請幫我整理成正式會議記錄（Task）：

[貼入逐字稿]

請依照以下格式輸出（Format）：
- 會議主旨：
- 出席人員（從逐字稿中推斷）：
- 討論重點（條列式）：
- 決議事項：
- 待辦事項表格：| 負責人 | 任務內容 | 完成期限 |

語言：繁體中文，語氣正式
```

---

**Hard — 練習 3：多版本語氣改寫**

```
題目：
原始句子：「那個功能昨天已經修好了，你們可以再試試看，應該沒問題了。」
請同時要求 AI 改寫成 3 種語氣：
① 正式客戶通知信語氣
② 親切服務語氣（給一般消費者）
③ 工程師交接語氣（給同事）

並說明三者的主要差異
```

**參考解答：**
```
請將以下文字分別改寫成三種不同語氣的版本（Task）：
原文：「那個功能昨天已經修好了，你們可以再試試看，應該沒問題了。」

版本一：正式客戶通知信語氣（收件對象：企業客戶，正式場合）

版本三：工程師交接語氣（收件對象：開發同事，Slack 訊息）

輸出後請說明三個版本在「用詞選擇」和「句子結構」上有何不同（Format）
```

### ✏️ 學習重點

> - 告訴 AI 「收件對象是誰」和「你們的關係」，語氣才會恰當
> - 需要多版本時，一次指令就要求全部，比分開問更有效率

## Unit 5 — 健康與醫療諮詢輔助

### 概念教學

**核心能力：** 用 AI 獲取健康資訊、管理用藥、整理報告，並清楚了解 AI 的限制。

> ⚠️ **核心警語：**  
> AI 提供的是「健康資訊參考」(Health Information Reference)，  
> 不是「醫療診斷」(Medical Diagnosis)。  
> 症狀嚴重或持續時，請務必就診。

**AI 健康助理的能力邊界（Boundaries）：**

```
✅ AI 可以做：                    ❌ AI 不應取代：
- 解釋醫學名詞                    - 確認診斷疾病
- 建議應掛哪一科                  - 調整處方藥物用量
- 設計用藥提醒計畫                - 判斷急症是否需要急診
- 整理健康檢查報告用語            - 解讀個人病史的複雜互動
- 提供一般預防保健知識
```

### 練習題

**Easy — 練習 1：症狀初步查詢**


## Unit 6 — 學習輔助與家教

### 概念教學

**核心能力：** 設計有效的 AI 學習對話，讓 AI 成為有耐心的個人家教。

**關鍵技術術語：**
- **Scaffolding**（鷹架教學）：從簡單到複雜的分層學習
- **Socratic Method**（蘇格拉底問答法）：透過反問引導思考
- **Retrieval Practice**（提取練習）：主動回憶比重複閱讀更有效

**有效學習互動的層次：**

```
Level 1：被動接收
→ 「解釋 XXX 是什麼」（效果：★★☆☆☆）

Level 2：主動理解
→ 「用比喻解釋 XXX，然後出一道例題讓我練習」（效果：★★★★☆）

Level 3：主動應用
→ 「先不要解釋，直接出題考我，我答完你再給解析」（效果：★★★★★）
```

## Unit 7 — 家庭財務管理

### 概念教學

**核心能力：** 把消費記錄轉化成可分析的數據，讓 AI 當你的財務顧問。

**關鍵技術術語：**
- **Cash Flow**（現金流）：收入與支出的流動情況
- **Savings Rate**（儲蓄率）：儲蓄佔可支配收入的比例
- **Budget Category**（預算分類）：將支出分組管理

**儲蓄率計算公式：**

$$\text{儲蓄率} = \frac{\text{月收入} - \text{月總支出}}{\text{月收入}} \times 100\%$$

**健康財務比例（50/30/20 法則）：**
```
月收入的 50% → 必要支出（房租、水電、食物）
月收入的 30% → 想要支出（娛樂、購物、餐廳）
月收入的 20% → 儲蓄與投資
```

---

### 練習題

**Easy — 練習 1：月度消費分析**

```

讓 AI 分類、計算並分析：

模擬數據（月薪 38,000 元，固定房租 12,000 元）：
外送食物：3,200 元、超商：980 元、Uber：1,500 元
衣服鞋子：4,200 元、電影票：480 元、書：350 元
健身房：1,200 元、藥局：320 元、聚餐：2,600 元

任務：讓 AI 計算儲蓄率，並找出超出 50/30/20 建議的項目
```

**參考解答：**
```

以下是我 5 月的消費記錄（Context）：
月薪：38,000 元
固定支出（未列入）：房租 12,000 元

可變支出：
- 外送食物：3,200 元
- 超商消費：980 元
- Uber 叫車：1,500 元
- 衣服鞋子：4,200 元
- 電影票：480 元
- 書籍：350 元
- 健身房月費：1,200 元
- 藥局：320 元
- 朋友聚餐：2,600 元

請幫我（Task）：
1. 將所有支出分類（必要/想要/儲蓄）
2. 計算本月儲蓄率（公式：（收入-支出）÷收入×100%）
3. 對照「50/30/20 法則」，指出哪類支出超標
4. 建議 3 個下個月可以節省的具體行動（Format）
```

```
請幫我（Task）：
```

## Unit 8 — 娛樂內容推薦

### 概念教學

**核心能力：** 讓 AI 理解你的「真正口味」，而非只推薦熱門排行榜。

**關鍵技術術語：**
- **Content-based Filtering**（內容篩選）：根據作品特性推薦相似內容
- **Preference Profile**（偏好檔案）：描述你的喜好模式
- **Negative Constraint**（負面約束）：明確說明不喜歡什麼

**精準推薦的關鍵：說「喜歡的原因」不只說「喜歡什麼」**

```
❌ 低品質輸入：「我喜歡《寄生上流》，推薦類似的電影」
↓ AI 只知道你喜歡這部電影，不知道原因

✅ 高品質輸入：「我喜歡《寄生上流》因為它有
   ① 深刻的社會階級批判 ② 出乎意料的劇情轉折 ③ 黑色幽默
   但我不喜歡純粹的暴力血腥場面，也不喜歡太正能量的結局」
↓ AI 能理解你的偏好維度，推薦更精準
```

---

### 練習題

**Easy — 練習 1：電影推薦（偏好維度法）**

```
題目：
選一部你最近看過且喜歡的電影，
用「偏好維度法」寫出推薦指令：
① 列出你喜歡這部電影的 3 個具體原因（情節/主題/風格）
② 說明 2 個你不喜歡的電影元素
③ 指定輸出格式（推薦 5 部，含簡介與觀看時機）
```

**範例參考解答：**
```
我最近看了《你的名字》並很喜歡（Context），原因是：
① 我喜歡它的：穿越時間的奇幻設定、視覺畫面的美感、讓人揪心的情感張力
② 我不喜歡：太過說教的道德故事、結局不清不楚的電影

請根據我的偏好推薦 5 部電影（Task），
每部附上（Format）：
- 電影名稱（年份）
- 一句話介紹（不爆雷）
- 與《你的名字》的相似點
- 適合什麼時候/什麼心情下看
```

**Medium — 練習 2：個人化音樂播放清單**

## Unit 9 — 客服與問題解決

### 概念教學

**核心能力：** 用 AI 把情緒化的抱怨轉化為理性且有力的客訴表達。

**關鍵技術術語：**
- **Escalation**（升級處理）：問題無法解決時，向上層反映
- **Consumer Rights**（消費者權益）：消費者保護法賦予的權利
- **Dispute Resolution**（爭議解決）：雙方解決糾紛的程序

**有效客訴的結構：**
```
① 事實陳述（FACT）：何時/何地/發生什麼事（不帶情緒）
② 影響說明（IMPACT）：這件事對你造成什麼損失或不便
③ 明確訴求（REQUEST）：你希望對方具體做什麼（退款/道歉/修正）
④ 時間限制（DEADLINE）：你希望何時得到回應
```

---

### 練習題

**Easy — 練習 1：撰寫投訴信**

```
題目：
```

## Unit 10 — 社群媒體與內容創作輔助

### 概念教學

**核心能力：** 讓 AI 產出符合你個人風格的社群內容，而非千篇一律的模板文。

**關鍵技術術語：**
- **Brand Voice**（品牌聲音）：你在社群上一貫的說話風格
- **Hook**（鉤子）：吸引受眾在前 3 秒停下來看的開頭
- **Call to Action, CTA**（行動呼籲）：引導讀者採取某個動作（留言/分享/點擊）

**提升 AI 文案品質的「風格鎖定法」：**
```
步驟 1：找出 3 篇你認為風格最接近的貼文（可以是你自己的或喜歡的帳號）
步驟 2：貼給 AI，說：「請分析這 3 篇文的風格特點（語氣/句長/用字習慣）」
步驟 3：AI 分析後，說：「請用這個風格幫我寫新的貼文」
```

---

### 練習題

**Easy — 練習 1：IG 照片文案**

```
題目：
你在花蓮太魯閣拍了一張峽谷照片，
天氣晴天，壯觀的峭壁與藍天。
你的帳號風格：文青旅遊記錄，偶爾帶點人生感悟。
```

```
- 天氣：晴天，光線強烈

請幫我寫 3 種不同風格的 IG 貼文文案（Task），各 80~120 字（Format）：
① 文青詩意風：有點哲思、有感悟、適合加 hashtag
```

```
② 根據分析，為我設計一份「品牌聲音指南」，包含：
```

## 學習路線圖

### AI Agent 使用能力學習階段

```
基礎期（第 1~2 週）
├── 了解 AI Agent 基本概念與限制
├── 學會 Role + Context + Task + Format 四要素指令結構
├── 練習情景 1~3（行程/購物/旅遊）的 Easy 題
└── 里程碑：能寫出完整的旅遊規劃指令並得到有用的回應

核心期（第 3~4 週）
├── 練習情景 4~7（文件/健康/學習/財務）
├── 學會多輪對話微調（Iterative Refinement）
├── 建立自己的「常用指令模板庫」
└── 里程碑：完成 Unit 1~7 的所有 Medium 題

進階期（第 5~6 週）
├── 練習情景 8~10（娛樂/客服/創作）
├── 學習 Prompt 審查（能自己發現指令的問題）
├── 開始建立個人 Prompt 模板庫（至少 10 個常用情景）
└── 里程碑：完成綜合測驗得到 80 分以上

實戰期（持續進行）
├── 每週選 1 個真實生活問題用 AI 解決
├── 紀錄「哪個指令有效/無效」並持續優化
├── 探索特定工具的進階功能（ChatGPT Actions/Claude Projects）
└── 里程碑：AI 成為你日常工作流程中的固定工具
```

### 每階段推薦練習平台
