## 1. 專案背景與目的

### 1.1 背景說明

本公司為模型玩具零售商，現有 classicmodels 資料庫中維護 **110 項產品**，涵蓋 7 條產品線，與 **13 家供應商**合作。目前採購作業缺乏系統化管理，以下問題亟待解決：

- 庫存預警機制不足，已出現部分商品庫存極低（最低僅 **15 件**）
- 採購單據以人工作業為主，易出現資料錯誤與遺漏
- 無法即時掌握各供應商報價與交貨績效
- 進價與建議售價（MSRP）缺乏動態比較與毛利分析

### 1.2 專案目的

建置一套「**產品採購管理系統**」，實現：

1. 自動化庫存監控與採購觸發機制
2. 電子化採購單建立、審核與追蹤流程
3. 多供應商報價比較與評估
4. 採購成本與毛利率即時分析
5. 完整的採購歷史紀錄與報表產出

### 採購作業流程

```
庫存低於安全水位
       │
       ▼
系統自動產生採購建議
       │
       ▼
採購人員建立採購申請單 ──► 主管審核 ──► 退回修改
       │                      │
       │                   核准通過
       │                      │
       ▼                      ▼
多供應商報價比較 ◄─────── 選擇供應商
       │
       ▼
建立採購訂單
       │
       ▼
發送訂單給供應商
       │
       ▼
供應商出貨 → 收貨驗收
       │
       ├── 合格 → 更新庫存 → 完成
       └── 不合格 → 退貨/補貨
```

## 8. 使用案例

### UC-01：自動庫存補貨觸發

**主要參與者：** 系統（排程任務）、採購人員

**前置條件：** 已設定各產品安全庫存水位

**主要流程：**
1. 系統每日 08:00 自動執行庫存掃描
2. 比對 `quantityInStock` 與 `stock_thresholds.minQuantity`
3. 找出低於安全水位的產品（如：S24_2000 庫存 15 件 < 安全水位 500 件）
4. 系統自動產生採購建議清單
5. 發送 Email 通知採購人員
6. 採購人員登入系統，確認採購建議後轉為採購申請單

---

### UC-02：多供應商比價採購

**主要參與者：** 採購人員、採購主管

**前置條件：** 採購申請單已核准

**主要流程：**
1. 採購人員依核准申請單，向至少 3 家供應商索取報價
2. 在系統中建立報價比較表
3. 系統計算各供應商的報價差異與歷史績效分數
4. 採購人員選定最優供應商
5. 系統自動建立採購訂單

---

### UC-03：收貨與庫存更新

**主要參與者：** 倉管人員

**前置條件：** 採購訂單狀態為「已發出」

**主要流程：**
1. 供應商貨品到達，倉管人員掃描（或手動輸入）採購訂單號
2. 逐項確認收貨數量與產品代碼
3. 登錄驗收結果（合格/不合格）
4. 系統自動更新 `products.quantityInStock`
5. 若全部收貨完成，採購訂單狀態更新為「完成」
6. 不合格品建立退貨申請單

## 9. 業務規則

### BR-01：庫存安全水位計算

- 預設安全水位 = 各產品線平均庫存 × 20%
- 各產品可由採購主管手動調整安全水位
- 安全水位計算參考值：

| 產品線 | 建議安全水位（件） |
|--------|------------------|
| Classic Cars | 1,000 |
| Vintage Cars | 800 |
| Motorcycles | 600 |
| Planes | 500 |
| Trucks and Buses | 500 |
| Ships | 400 |
| Trains | 300 |

### BR-02：採購申請審核規則

| 申請金額 | 審核層級 |
|----------|----------|
| ≤ $5,000 | 採購主管一級審核 |
| $5,001 ~ $20,000 | 採購主管 + 財務主管 |
| > $20,000 | 採購主管 + 財務主管 + 總經理 |

### BR-03：毛利保護規則

- 採購進價（buyPrice）不得超過建議售價（MSRP）的 60%
- 若報價超過此上限，系統應顯示警示並要求主管核准
- 目前平均毛利率為 **88.7%**，應維護於 **80% 以上**

### BR-04：供應商評分規則

| 評分項目 | 權重 | 評分說明 |
|----------|------|----------|
| 交期達成率 | 40% | 準時交貨次數 / 總交貨次數 |
| 品質合格率 | 40% | 合格收貨數量 / 總收貨數量 |
| 價格競爭力 | 20% | 與市場均價比較 |

- 供應商總分 < 60 分時，系統標記為「需輔導供應商」
- 供應商總分 < 40 分時，系統自動暫停向該供應商採購

### BR-05：採購訂單規則

- 採購訂單一旦「已發出」，不得直接刪除，只能「取消」
- 取消訂單須記錄取消原因與操作人員
- 部分收貨時，未收貨部分可選擇「繼續等待」或「建立補單」
