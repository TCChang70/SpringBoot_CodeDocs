# 供應商管理系統 - 初學者學習指南

## 📖 學習目標
通過本教學，您將學會：
- 理解 RESTful Web Service 的基本概念
- 學習 JAX-RS 框架的使用方法
- 掌握 JPA (Java Persistence API) 的資料庫操作
- 了解 MVC 架構模式的實際應用

## 🏗️ 系統架構概覽

本系統採用三層式架構：
```
SupplierService (控制層) 
    ↓
SupplierDAO (資料存取層)
    ↓
Supplier (資料模型層)
```

## 📁 檔案結構說明

### 1. Supplier.java - 資料模型類 (Entity)
```java
@Entity
@Table(name="suppliers")
public class Supplier implements Serializable
```

**關鍵註解說明：**
- `@Entity`：標記此類為 JPA 實體類，對應資料庫中的一張表
- `@Table(name="suppliers")`：指定對應的資料庫表名為 "suppliers"
- `@Id`：標記主鍵欄位
- `@Column(name="SUP_ID")`：指定對應的資料庫欄位名

**屬性對應：**
| Java 屬性 | 資料庫欄位 | 說明 |
|-----------|------------|------|
| supId | SUP_ID | 供應商編號（主鍵）|
| supName | SUP_NAME | 供應商名稱 |
| city | city | 城市 |
| state | state | 州/省 |
| street | street | 街道 |
| zip | zip | 郵遞區號 |

### 2. SupplierDAO.java - 資料存取物件

**主要功能：**
- 管理資料庫連線
- 執行 CRUD（增刪改查）操作

#### 核心方法解析：

##### 🔗 建立資料庫連線
```java
public EntityManager createConnection() {
    EntityManagerFactory factory = Persistence.createEntityManagerFactory("mvjpa1104");
    EntityManager mgr = factory.createEntityManager();
    return mgr;
}
```
- `EntityManagerFactory`：JPA 的工廠類，用於建立 EntityManager
- `EntityManager`：JPA 的核心介面，負責資料庫操作

##### 📋 查詢所有供應商
```java
public List<Supplier> getAll() {
    TypedQuery<Supplier> query = mgr.createNamedQuery("Supplier.findAll", Supplier.class);
    return query.getResultList();
}
```
- 使用命名查詢 `Supplier.findAll`
- 返回所有供應商的列表

##### 🔍 根據ID查詢供應商
```java
public Supplier findById(int id) {
    String sql = "select e from Supplier e where e.supId = :arg1";
    // ... 查詢邏輯
}
```
- 使用 JPQL（Java Persistence Query Language）
- `:arg1` 是參數佔位符

##### ➕ 新增供應商
```java
public boolean addSupplier(Supplier s1) {
    mgr.getTransaction().begin();  // 開始交易
    mgr.persist(s1);               // 持久化物件
    mgr.getTransaction().commit(); // 提交交易
}
```

##### ✏️ 更新供應商
```java
public boolean updateSupplier(int id, Supplier s1) {
    Supplier found = mgr.find(Supplier.class, id);  // 先查詢
    mgr.merge(s1);  // 合併更新
}
```

##### ❌ 刪除供應商
```java
public boolean deleteSupplier(int id) {
    Supplier found = mgr.find(Supplier.class, id);  // 先查詢
    mgr.remove(found);  // 移除物件
}
```

### 3. SupplierService.java - RESTful Web Service

這是系統的控制層，提供 HTTP API 接口。

#### RESTful API 設計：

| HTTP 方法 | URL 路徑 | 功能 | 範例 |
|-----------|----------|------|------|
| GET | /suppliers | 取得所有供應商 | `GET /suppliers` |
| GET | /suppliers/{id} | 取得指定供應商 | `GET /suppliers/1` |
| POST | /suppliers | 新增供應商 | `POST /suppliers` |
| PUT | /suppliers/{id} | 更新供應商 | `PUT /suppliers/1` |
| DELETE | /suppliers/delete/{id} | 刪除供應商 | `DELETE /suppliers/delete/1` |

#### 重要註解說明：

##### 類別層級註解
```java
@Path("/suppliers")
public class SupplierService
```
- `@Path("/suppliers")`：定義基礎 URL 路徑

##### 方法層級註解
```java
@GET
@Produces(MediaType.APPLICATION_JSON)
public List<Supplier> getAll()
```
- `@GET`：處理 HTTP GET 請求
- `@Produces`：指定回應的資料格式（JSON）
- `@Consumes`：指定接收的資料格式（JSON）
- `@PathParam("id")`：從 URL 路徑中提取參數

#### HTTP 狀態碼使用：

```java
return Response
    .status(Response.Status.OK)      // 200 - 成功
    .entity(s1).build();

return Response
    .status(Response.Status.NOT_FOUND)    // 404 - 找不到資源
    .build();

return Response
    .status(Response.Status.BAD_REQUEST)  // 400 - 請求錯誤
    .build();
```

## 🔧 開發環境設定

### 必要依賴 (Maven)
```xml
<dependencies>
    <!-- JAX-RS 實作 -->
    <dependency>
        <groupId>org.glassfish.jersey.core</groupId>
        <artifactId>jersey-server</artifactId>
        <version>2.32</version>
    </dependency>
    
    <!-- JPA 實作 -->
    <dependency>
        <groupId>org.hibernate</groupId>
        <artifactId>hibernate-entitymanager</artifactId>
        <version>5.4.32.Final</version>
    </dependency>
    
    <!-- JSON 處理 -->
    <dependency>
        <groupId>org.glassfish.jersey.media</groupId>
        <artifactId>jersey-media-json-jackson</artifactId>
        <version>2.32</version>
    </dependency>
</dependencies>
```

### persistence.xml 配置
```xml
<?xml version="1.0" encoding="UTF-8"?>
<persistence xmlns="http://java.sun.com/xml/ns/persistence" version="2.0">
    <persistence-unit name="mvjpa1104">
        <provider>org.hibernate.ejb.HibernatePersistence</provider>
        <properties>
            <property name="javax.persistence.jdbc.driver" value="com.mysql.cj.jdbc.Driver"/>
            <property name="javax.persistence.jdbc.url" value="jdbc:mysql://localhost:3306/supplier_db"/>
            <property name="javax.persistence.jdbc.user" value="root"/>
            <property name="javax.persistence.jdbc.password" value="password"/>
            <property name="hibernate.hbm2ddl.auto" value="update"/>
            <property name="hibernate.show_sql" value="true"/>
        </properties>
    </persistence-unit>
</persistence>
```
