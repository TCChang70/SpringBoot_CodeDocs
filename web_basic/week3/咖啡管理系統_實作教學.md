

# 咖啡管理系統（Coffee）- 初學者教學文件
========================================

## 學習目標
- 了解如何以 JAX-RS 建立 RESTful API
- 掌握 JPA 在資料庫 CRUD 操作中的使用方式
- 熟悉三層式架構（Service / DAO / Entity）在實務專案的應用


## 系統架構概覽

```
CoffeeService (控制層 / REST API)
        ↓
CoffeeDAO (資料存取層 / JPA)
        ↓
Coffee (資料模型層 / Entity)
```

- **控制層**：負責接收 HTTP 請求並回傳結果
- **資料存取層**：封裝與資料庫的互動（JPA）
- **資料模型層**：描述資料表欄位與 Java 物件的對應

## 主要類別與責任分工

### 1. `Coffee` 實體（Entity）

```java
@Entity
@Table(name = "coffees")
@NamedQuery(name = "Coffee.findAll", query = "SELECT c FROM Coffee c")
public class Coffee implements Serializable {
    @Id
    @Column(name = "COF_NAME")
    private String cofName;   // 主鍵：咖啡名稱

    private BigDecimal price; // 價格
    private int sales;        // 銷售量

    @Column(name = "SUP_ID")
    private int supId;        // 供應商 ID（對應 Supplier.supId）

    private int total;        // 庫存或總量
}
```

| Java 屬性 | 資料庫欄位  |              說明                  |
|-----------|------------|------------------------------------|
| cofName   | COF_NAME   | 主鍵，咖啡名稱                      |
| price     | price      | 單價                               |
| sales     | sales      | 銷售量                             |
| supId     | SUP_ID     | 供應商編號，建議與 Supplier 建立關聯 |
| total     | total      | 庫存數量                           |

> 提醒：主鍵為字串，所有 API 的 Path 參數需使用字串型別。

### 2. `CoffeeDAO`（資料存取層）

- 建立與關閉 `EntityManager`
- 透過 JPA 執行查詢與交易
- 封裝 CRUD 方法供 Service 呼叫

常見方法：

```java
public List<Coffee> getAll();
public Coffee findByName(String cofName);
public boolean addCoffee(Coffee c1);
public boolean updateCoffee(String cofName, Coffee c1);
public boolean deleteCoffee(String cofName);
```

關鍵實作重點：
- 交易需以 `getTransaction().begin()` 與 `commit()` 包起來
- 發生例外時記得 `rollback()` 並回傳 `false`
- 更新時需確認主鍵一致，例如 `c1.setCofName(cofName);`
- 刪除前務必確認資料存在，避免 `NullPointerException`

### 3. `CoffeeService`（控制層 / REST API）

- 負責定義 API 路徑與 HTTP 方法
- 取得輸入參數並呼叫 DAO 執行邏輯
- 組合適當的 HTTP 狀態碼與回應內容

常見註解：
- `@Path`：設定 URI 路徑
- `@GET`、`@POST`、`@PUT`、`@DELETE`：對應 HTTP 動詞
- `@Produces(MediaType.APPLICATION_JSON)`：回傳 JSON
- `@Consumes(MediaType.APPLICATION_JSON)`：接受 JSON 請求
- `@PathParam("name")`：從 path 取得參數

## REST API 設計

| HTTP 方法 |      路徑               |    功能說明    |        範例 |
|----------|-------------------------|----------------|---------------------|
| GET      | `/coffees`              | 取得所有咖啡資料| `GET /coffees` |
| GET      | `/coffees/{name}`       | 依名稱查詢咖啡  | `GET /coffees/Colombian` |
| POST     | `/coffees`              | 新增咖啡資料    | `POST /coffees` |
| PUT      | `/coffees/{name}`       | 更新咖啡資料    | `PUT /coffees/Colombian` |
| DELETE   | `/coffees/delete/{name}`| 刪除咖啡資料 | `DELETE /coffees/delete/Colombian` |

API 回應範例：

```java
return Response.status(Response.Status.OK).entity(c1).build();      // 200 成功
return Response.status(Response.Status.NOT_FOUND).build();          // 404 找不到
return Response.status(Response.Status.BAD_REQUEST).build();        // 400 請求錯誤
```

## API 測試範例

### 1. 取得全部咖啡
```
GET /coffees
Accept: application/json
```

### 2. 查詢特定咖啡
```
GET /coffees/Latte
Accept: application/json
```

### 3. 新增咖啡
```
POST /coffees
Content-Type: application/json

{
    "cofName": "Latte",
    "price": 120.0,
    "sales": 50,
    "supId": 1,
    "total": 200
}
```

### 4. 更新咖啡
```
PUT /coffees/Latte
Content-Type: application/json

{
    "cofName": "Latte",
    "price": 130.0,
    "sales": 80,
    "supId": 1,
    "total": 180
}
```

### 5. 刪除咖啡
```
DELETE /coffees/delete/Latte
```

## 開發環境與設定

### Maven 依賴（節錄）

```xml
<dependency>
    <groupId>org.glassfish.jersey.core</groupId>
    <artifactId>jersey-server</artifactId>
    <version>2.32</version>
</dependency>
<dependency>
    <groupId>org.glassfish.jersey.media</groupId>
    <artifactId>jersey-media-json-jackson</artifactId>
    <version>2.32</version>
</dependency>
<dependency>
    <groupId>org.hibernate</groupId>
    <artifactId>hibernate-entitymanager</artifactId>
    <version>5.4.32.Final</version>
</dependency>
```

### `persistence.xml` 設定重點
- `persistence-unit` 名稱需與 DAO 中一致（此範例為 `mvjpa1104`）
- 設定資料庫連線資訊（JDBC URL、帳號、密碼）
- 可啟用 `hibernate.show_sql` 以方便除錯

## 學習重點整理

### 架構與設計
- 套用 DAO Pattern 分離資料存取邏輯
- 實作 RESTful API 並正確使用 HTTP 方法
- 使用 Entity 管理資料表欄位映射

### 技術細節
- JPA 交易管理與例外處理（commit / rollback）
- 使用 `NamedQuery` 與 `JPQL` 查詢
- JSON 序列化與 `MediaType.APPLICATION_JSON`

### 最佳實務
- 新增與更新前先驗證資料完整性
- 刪除前先檢查目標是否存在
- 完成操作後適當釋放資源（EntityManager）
- 為 API 撰寫單元測試或整合測試

## 進階延伸建議
1. 使用 Bean Validation (`@NotNull`, `@Size`, `@DecimalMin` 等) 進行欄位驗證
2. 與 `Supplier` 建立關聯（`@ManyToOne`、`@JoinColumn`）
3. 加入分頁、排序與條件查詢功能
4. 透過 Swagger/OpenAPI 產生 API 文件
5. 撰寫整合測試（例如使用 Jersey Test 或 REST Assured）
